核心功能引导

5步遮罩引导，覆盖 Tables / Club / Games 三个模块
▼
必做 Tables页 — 3步引导

Step1 高亮 CREATE → "这里可以免费创建房间，将房间码分享给好友即可开局"

Step2 高亮 JOIN → "收到好友的房间码？点这里输入，快速加入对局"

Step3 高亮 Club图标 → "这里是俱乐部入口，加入好友的俱乐部或创建自己的俱乐部"

触发时机：注册奖励弹窗关闭回调后（事件驱动，非固定延迟）

开发成本：1-2天

必做 Club页 — 1步引导

高亮 "Enter Club ID" 输入框

"向好友获取俱乐部ID，输入申请加入。审批通过后会通知你"

触发时机：首次切换到Club页签，页面加载完成后

开发成本：0.5天

必做 Games页 — 1步引导

高亮主推游戏卡片

"这里是钻石小游戏专区，参与游戏需消耗钻石"

触发时机：首次切换到Games页签，页面加载完成后

开发成本：0.5天

🎭

身份建立

个性化设置 · 沉没成本效应 · 为分享卡片提供素材
▼
优化 引导完成后个性化设置

选择头像（3-5个预设角色）+ 设置昵称 + 选择常玩玩法

作用：投入时间做个性化 → 降低离开意愿（沉没成本）

附加：后续分享卡片展示头像和昵称，让分享更有"人味"

开发成本：1天

当前不做：

新手教学牌局（无PVP大厅承载）/ AI对战练习模式（开发成本高）/ 复杂的用户画像问卷（DAU太低没统计意义）

② 引导到分享

3-5分钟 · 把"没人玩"的困境转化为"我要去拉人"的行动
🎯

首日任务面板

核心转化机制 · 给用户明确的目标链
▼
必做 任务+奖励链

✅ 完成注册 → +500 💎

⬜ 邀请1位好友 → +1000 💎

⬜ 完成第一局牌局 → +2000 💎

🏆 全部完成额外奖励 → 限定头像框

核心逻辑：把"拉人来玩"变成一个有奖励、有进度条、有目标感的任务

"立即邀请好友"作为唯一主按钮，所有UI设计指向一个动作：去拉人

开发成本：2-3天（任务系统 + UI + 奖励发放）

🤝

邀请奖励机制

双向激励 · 邀请者和被邀请者都有收益
▼
重要 邀请双方奖励

邀请者：每邀请1人注册 → +1000 💎

被邀请者：通过邀请链接注册 → 额外注册奖励加成

作用：邀请者有动力持续拉人，被邀请者有额外入驻动力

需要后端支持邀请关系追踪（短期可用邀请码手动绑定）

开发成本：2天（邀请码体系 + 奖励发放）

当前不做：

多级邀请返利（容易被刷）/ 排行榜竞赛邀请（DAU太低）/ 付费邀请广告投放（先跑通自然裂变）

③ 分享体验

核心中的核心 · 让"邀请好友"这个动作尽可能简单、有面子、有效
🎴

分享卡片与渠道

可视化邀请 + 系统级分享面板
▼
必做 可视化邀请卡片

包含：用户头像+昵称 / 房间码 / 玩法和盲注配置 / 下载链接

vs 现状（纯文字房间码）：

  • 有"面子"——正式的邀请，不是一串冷冰冰的代码
  • 信息完整——朋友知道这是什么，降低决策成本
  • 有下载入口——新用户有明确行动路径

开发成本：2天（卡片模板 + 动态内容填充）

必做 系统分享面板

调起 iOS Share Sheet / Android Share Intent

一键分享到 WhatsApp / Telegram / LINE / Facebook Messenger

兜底：复制链接按钮（无社交工具时）

开发成本：1天（系统API调用）

必做 一键建房+分享

点击"邀请好友" → 自动以默认配置创建房间 → 直接调起分享面板

把5步操作压缩成1步：创建→配置→复制码→切出游戏→发给朋友 → 点一下

开发成本：1天

📍

多场景分享入口

不只是房间内能分享 · 处处可分享
▼
重要 分享入口布局
  • 创建房间后：立即弹出分享面板（最高优先级入口）
  • 房间内：常驻分享按钮
  • Tables空状态："创建房间并邀请好友"主按钮
  • 首日任务面板："立即邀请好友"按钮
  • Profile页：个人邀请码/链接展示

开发成本：1.5天

当前不做：

分享到社群自动建群（功能过重）/ 分享视频/GIF精彩回放（需要录制系统）/ SNS广告素材自动生成

④ 等待与召回

分享之后 · 好友没马上来的空窗期 · 防止等待流失
⏳

等待期体验

分享后不要让用户盯着空房间
▼
重要 等待房间状态页

分享成功后进入等待状态：

  • 展示房间码 + "再次分享"按钮（发了一个人没来？再发给另一个）
  • "好友加入后自动通知你" → 管理预期
  • 等待期可做的事：浏览俱乐部 / 试玩Games小游戏
  • 可退出等待 → 好友加入后Push通知拉回

开发成本：2天

🔔

Push通知体系

关键时刻把用户拉回来
▼
重要 分阶段推送策略

T+1h 好友未加入 → "你的好友还没加入，再邀请一位朋友试试？"

T+Xh 好友加入房间 → "好友已加入你的房间，快来开局！"

T+Xh 好友注册了 → "你的好友XX已注册，邀请TA加入房间吧！"

T+24h 任务未完成 → "你的首日任务还差一步，今天完成可获2000💎"

开发成本：1.5天（Push通道 + 触发逻辑）

重要 俱乐部审批通知

管理者审批通过 → Push通知 + 游戏内红点

进入俱乐部后，如有正在进行的牌局 → 引导加入

审批等待期是高流失节点，必须做好通知

开发成本：1天

空状态优化

Tables页最高频停留页面 · 从"告知"变为"推动"
▼
重要 Tables空状态改版

现状："No active games. Create one?" + 一个灰色按钮

改为：

  • [创建房间并邀请好友] — 主按钮（突出、带颜色）

  • 💡 Tip："分享房间码给好友，即可开局"

核心变化：空状态从被动的"没有游戏"变为主动的"去创建和邀请"
